一、MVP 范围

做什么 · 不做什么

做不做
单用户本地；Agent 爬取不简化多用户/权限
Agent 推断本体；本体驱动展示拖拽本体编辑器
Action & Function 基础复杂工作流、批量/审计
NL 查询；关系图谱仪表盘、时间线、本体版本管理
验证场景

公开政治人物关系网络

典型流：自然语言意图 → Agent 搜索/爬取 → 结构化提取 → 推断本体 → 用户确认 → 列表/图谱/NL 深挖。

  • 例：整理特朗普核心圈；问「经济政策顾问是谁？」；梳理马斯克—特朗普关系链
  • Action：标记鹰派、生成关系简报等（后期阶段）
场景边界 · 选型理由

边界

白名单：Wikipedia、官方页、公开人物介绍、少量媒体；仅公开可验证信息；不争辩主观立场；争议关系进人工确认。

为何选此场景

数据公开利于快验主链路；关系复杂能展示本体+图谱；分析师有真实需求；演示叙事清晰。

二、技术架构（精简）

Next.js（对话 / 对象浏览 / D3·force-graph） ⟷ FastAPI：Ontology Engine + Agent Engine + Action/Function Runtime + NL Interface；Redis + Celery 异步采集；PostgreSQL 单库（JSONB + 递归 CTE，MVP 不引入 Neo4j）。

三、自然语言查询

① 结构化过滤

例：「共和党内阁成员」→ LLM 转条件 → 属性查询。MVP 必做

② 语义相似

向量化 + pgvector 检索。可选增强（视 P3 进度）

③ 图关系

一跳/多跳、路径 → 图遍历。MVP 必做

四、可信机制（必做）

非「抓完即真」—— 自动提取 + 可追溯 + 可审核。

  • 元数据：来源 URL/站点、抓取时间、置信度、人工确认、原始片段
  • 管道：raw_sources → 候选 entity_candidates → 确认 → objects/links
  • 冲突：不自动覆盖高可信已确认；争议进 review_queue；展示「采用值 + 来源」
审核队列 UI（P1 必做）

高效、少填表

  • 列表 + 详情分栏；筛选：低置信 / 冲突 / 新实体
  • 一键确认/拒绝；冲突字段并排高亮 — 选策略而非手写
  • 快捷键：J/K 导航，Y/N 通过/拒绝
五、Prompt 管理

prompts/ 分 agent / nl_interface / action；YAML：name、version、model、temperature、system、user_template、few_shots。

  • 迭代 递增版本（v1→v2），旧版保留；验证集对比后再切生产
  • 代码引用指定版本，便于回滚
六 · P1 地基（第 1–2 周）

能存、能看、能审核

  • PostgreSQL Schema；Ontology API（types / objects / links / 查询与详情）
  • Redis+Celery 打底；前端：对象列表、详情（属性+关系+来源）、审核队列
  • 种子数据 10–20 人手录特朗普圈，跑通闭环
  • 验收：手动建模浏览可见来源；队列可确认/拒绝
P2 Agent 大脑（第 3–5 周） ⭐

自动采集 → 候选 → 确认入库

  • 规划器、白名单提取（静态/Playwright）、本体建议、清洗去重、候选管道
  • API：POST /agent/collect、任务进度、确认推断
  • 前端：意图输入、流式进度、推断确认、结果与来源摘要
  • 验收：一句「收集内阁成员」可走通白名单采集与入库
P3（第 6–7 周）

图谱 + 自然语言问答

  • 关系图：中心展开、点击下层、类型配色、缩放拖拽搜索
  • NL：属性过滤；一跳/两跳；基础路径；结果卡片与迷你图
  • 验收：核心网络可视；过滤与关系类问题答对
P4 · P5

P4（8–9 周）能操作

Action：打标签、关系简报；Function：路径 ≤3 跳、影响力排序；NL + 按钮触发。

验收：NL 打标签/简报；路径与 Top 影响力可查。

P5（第 10 周）打磨演示

扩充 50–100 人、关系与标签；流式 Agent、图谱与性能；3 分钟演示脚本；政治人物本体模板沉淀。

八、技术风险
风险应对
反爬封锁友好源优先；间隔；备用数据源
LLM 提取不准人工审核；few-shot；保留 raw 可追溯
Playwright 复杂度MVP 先静态；JS 站点按需引入
争议数据伤信任白名单；争议不入库；展示来源与确认态
大图卡顿单次 ≤200 节点；分层加载
单人进度严格砍范围不砍质量；AI 辅助编码
九、评估 · 验证集 · 回退
MVP 指标目标
单次采集任务成功率≥ 70%
人物基础属性准确率≥ 80%
关系识别准确率≥ 70%
候选人工确认通过率≥ 60%
意图 → 首次可浏览≤ 3 分钟
图谱首屏（≤100 节点）≤ 2 秒

验证集：20–30 标准人物 + 基准字段；每周抽样对比 Prompt 版本。回退：若第5周前全自动不稳 → 系统给建议 URL → 用户确认 URL → Agent 仅对确认 URL 提取 → 再审核入库。
